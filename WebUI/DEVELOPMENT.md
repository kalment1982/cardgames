## 🚀 如何运行

### 方法1：直接运行
```bash
cd WebUI
dotnet run
```
然后访问：http://localhost:5000

### 方法2：开发模式（热重载）
```bash
cd WebUI
dotnet watch run
```

### 方法3：发布生产版本
```bash
cd WebUI
dotnet publish -c Release
```

## 🐛 已修复的问题

1. **项目引用问题**
   - 问题：TractorGame.csproj包含了WebUI的文件
   - 解决：添加排除规则，排除WebUI目录

2. **编译错误**
   - 问题：AIPlayerTests.cs中Assert.All和Sum方法使用错误
   - 解决：修正语法，添加using System.Linq

3. **命名空间缺失**
   - 问题：GamePage.razor缺少TractorGame.Core.Rules命名空间
   - 解决：添加@using指令

4. **图片路径错误**
   - 问题：图片路径使用了相对路径../
   - 解决：修正为assets/开头的路径

## 📝 注意事项

1. **浏览器兼容性**
   - 推荐Chrome、Safari、Edge
   - 需要支持WebAssembly
   - 需要支持CSS3

2. **性能考虑**
   - 首次加载需要下载WebAssembly（约2-3MB）
   - 建议使用Release模式发布
   - 可以启用Brotli压缩

3. **开发建议**
   - 使用dotnet watch run进行开发
   - 使用浏览器开发者工具调试
   - 注意Blazor的生命周期
