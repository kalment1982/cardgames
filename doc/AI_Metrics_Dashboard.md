# AI管理指标仪表板设计

## 1. 指标分层架构

```
┌─────────────────────────────────────────────────┐
│  L1: 实时运营指标（Real-time Operations）        │
│  目的：快速发现问题，触发告警                      │
│  频率：每局/每小时                                │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  L2: 业务健康指标（Business Health）             │
│  目的：评估用户体验，指导产品决策                  │
│  频率：每日/每周                                  │
└─────────────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────────────┐
│  L3: 进化质量指标（Evolution Quality）           │
│  目的：评估自主进化系统效果                       │
│  频率：每代/每月                                  │
└─────────────────────────────────────────────────┘
```

## 2. L1 实时运营指标（Operations KPIs）

### 2.1 硬约束监控（红线指标）

| 指标ID | 指标名称 | 计算公式 | 阈值 | 告警级别 |
|-------|---------|---------|------|---------|
| OP-01 | 合法性率 | `(合法决策数 / 总决策数) * 100%` | =100% | P0-立即回滚 |
| OP-02 | P99延迟达标率 | `P99(决策延迟) < 150ms` | >99% | P1-紧急 |
| OP-03 | 平均延迟达标率 | `Avg(决策延迟) < 100ms` | >99% | P1-紧急 |
| OP-04 | 崩溃率 | `(崩溃局数 / 总局数) * 100%` | =0% | P0-立即回滚 |
| OP-05 | 内存溢出率 | `(内存>50MB的采样数 / 总采样数)` | =0% | P1-紧急 |

**监控窗口**: 滚动1小时
**采样频率**: 每局记录
**告警触发**: 任一指标违反阈值立即触发

### 2.2 性能监控（黄线指标）

| 指标ID | 指标名称 | 计算公式 | 目标值 | 警告阈值 |
|-------|---------|---------|-------|---------|
| OP-11 | 平均决策延迟 | `Avg(决策耗时)` | <80ms | >90ms |
| OP-12 | P95延迟 | `P95(决策耗时)` | <120ms | >130ms |
| OP-13 | 内存占用中位数 | `Median(内存占用)` | <35MB | >45MB |
| OP-14 | CPU占用峰值 | `Max(CPU%)` | <8% | >12% |

**监控窗口**: 滚动24小时
**告警触发**: 连续3小时超过警告阈值

### 2.3 稳定性监控

| 指标ID | 指标名称 | 计算公式 | 目标值 | 说明 |
|-------|---------|---------|-------|------|
| OP-21 | 胜率波动系数 | `StdDev(每日胜率) / Mean(每日胜率)` | <0.05 | 检测策略漂移 |
| OP-22 | 决策分布熵 | `H = -Σ(p_i * log(p_i))` | >2.5 | 策略多样性 |
| OP-23 | 异常决策率 | `(超时+重试+异常) / 总决策数` | <0.1% | 系统健康度 |

## 3. L2 业务健康指标（Business Health KPIs）

### 3.1 用户体验指标

| 指标ID | 指标名称 | 计算公式 | 目标范围 | 业务意义 |
|-------|---------|---------|---------|---------|
| BH-01 | 玩家完局率 | `(完成局数 / 开始局数) * 100%` | >85% | 游戏吸引力 |
| BH-02 | 重复对局率 | `(玩N局以上玩家 / 总玩家)` | >60% | 留存意愿 |
| BH-03 | 难度切换率 | `(切换难度次数 / 总玩家)` | <15% | 难度匹配度 |
| BH-04 | 平均游戏时长 | `Avg(局时长)` | 8-12分钟 | 节奏合理性 |

### 3.2 游戏质量指标

| 指标ID | 指标名称 | 计算公式 | 目标范围 | 说明 |
|-------|---------|---------|---------|------|
| BH-11 | 接近局比率 | `(分差<20分的局数 / 总局数)` | 40-60% | 竞争激烈度 |
| BH-12 | 逆转局比率 | `(中途落后但获胜局数 / 总局数)` | 20-30% | 戏剧性 |
| BH-13 | 碾压局比率 | `(分差>60分的局数 / 总局数)` | <10% | 避免挫败感 |
| BH-14 | 平均回合数 | `Avg(每局回合数)` | 10-14回合 | 游戏节奏 |

### 3.3 难度梯度健康度

| 难度 | 指标ID | vs对象 | 目标胜率 | 当前值 | 状态 |
|-----|-------|--------|---------|-------|------|
| Easy | BH-21 | 真人新手 | 65-75% | [待填] | ⚪ |
| Medium | BH-22 | 真人普通 | 45-55% | [待填] | ⚪ |
| Hard | BH-23 | 真人熟练 | 35-45% | [待填] | ⚪ |
| Expert | BH-24 | 真人高手 | 25-35% | [待填] | ⚪ |

**玩家分层标准**:
- 新手: 总局数<20 或 近期胜率<40%
- 普通: 总局数20-100 且 胜率40-60%
- 熟练: 总局数>100 且 胜率50-70%
- 高手: 总局数>200 且 胜率>65%

### 3.4 AI人性化指标

| 指标ID | 指标名称 | 计算公式 | 目标范围 | 说明 |
|-------|---------|---------|---------|------|
| BH-31 | 最优决策率 | `(专家标注最优决策数 / 总决策数)` | 85-95% | 不追求100% |
| BH-32 | 明显错误率 | `(明显错误数 / 关键决策数)` | <5% | Easy可放宽到<10% |
| BH-33 | 决策时间波动 | `StdDev(决策延迟) / Mean(决策延迟)` | >0.2 | 模拟思考 |
| BH-34 | 策略风格一致性 | `相同局面相同决策概率` | >0.6 | 有个性 |

## 4. L3 进化质量指标（Evolution Quality KPIs）

### 4.1 进化效率指标

| 指标ID | 指标名称 | 计算公式 | 目标值 | 说明 |
|-------|---------|---------|-------|------|
| EV-01 | 晋升成功率 | `(晋升代数 / 总代数) * 100%` | >20% | 每5代至少1次 |
| EV-02 | 代际耗时 | `Avg(单代评估时长)` | <3小时 | 分层评估后 |
| EV-03 | 回滚率 | `(回滚次数 / 晋升次数) * 100%` | <10% | 门禁有效性 |
| EV-04 | 停滞代数 | `连续无晋升的代数` | <8代 | 触发探索增强 |

### 4.2 进化方向指标

| 指标ID | 指标名称 | 计算公式 | 趋势要求 | 说明 |
|-------|---------|---------|---------|------|
| EV-11 | 累计胜率提升 | `(当前Champion胜率 - Baseline胜率)` | 单调递增 | 相对v0基线 |
| EV-12 | 明显错误率改善 | `(Baseline错误率 - 当前错误率)` | 单调递增 | 质量提升 |
| EV-13 | 策略多样性 | `H = -Σ(p_i * log(p_i))` | 保持>2.5 | 避免过拟合 |
| EV-14 | Elo评分 | `动态Elo计算` | 单调递增 | 绝对实力 |

### 4.3 安全性指标（零容忍）

| 指标ID | 指标名称 | 阈值 | 触发动作 |
|-------|---------|------|---------|
| EV-21 | 规则违反次数 | =0 | 立即淘汰候选 |
| EV-22 | 信息泄漏检测 | =0 | 立即淘汰+审计 |
| EV-23 | 性能退化检测 | 延迟增幅<20% | 晋升门禁拦截 |
| EV-24 | 内存泄漏检测 | 增长率<5% | 晋升门禁拦截 |

### 4.4 代际对比指标

每代生成对比报告，包含：

```markdown
## Generation N vs Generation N-1

### 核心指标对比
| 指标 | Gen N-1 | Gen N | 变化 | 显著性 |
|-----|---------|-------|------|--------|
| 胜率 | 52.3% | 54.1% | +1.8% | p<0.01 ✅ |
| 明显错误率 | 4.2% | 3.8% | -0.4% | p<0.05 ✅ |
| P99延迟 | 142ms | 138ms | -4ms | - |

### 置信区间
- 胜率提升95% CI: [+0.9%, +2.7%]
- 下界>0，满足晋升条件 ✅

### 稳定性验证
- 3次重复评估方差: 0.8%（<2%阈值）✅
```

## 5. 指标采集与存储

### 5.1 数据流设计

```
游戏对局
    ↓
[实时采集] → 决策日志（每决策）
    ↓
[聚合层] → 小时级/日级指标
    ↓
[存储层] → 时序数据库（InfluxDB/Prometheus）
    ↓
[展示层] → Grafana仪表板
```

### 5.2 日志格式规范

```json
{
  "timestamp": "2026-03-14T10:23:45Z",
  "game_id": "g_20260314_001234",
  "round": 5,
  "player_id": "ai_hard_v1.2",
  "decision_type": "follow",
  "decision_latency_ms": 67,
  "memory_mb": 34.2,
  "is_legal": true,
  "is_optimal": true,
  "is_blunder": false,
  "cards_played": ["♠A", "♠A"],
  "context": {
    "hand_size": 18,
    "trump_suit": "♠",
    "is_leading": false
  }
}
```

### 5.3 聚合规则

```sql
-- 示例：计算小时级P99延迟
SELECT
  time_bucket('1 hour', timestamp) AS hour,
  player_id,
  percentile_cont(0.99) WITHIN GROUP (ORDER BY decision_latency_ms) AS p99_latency
FROM decision_logs
WHERE timestamp > NOW() - INTERVAL '24 hours'
GROUP BY hour, player_id;
```

## 6. 告警规则配置

### 6.1 P0告警（立即回滚）

```yaml
- alert: IllegalMoveDetected
  expr: sum(rate(illegal_moves_total[5m])) > 0
  for: 0m
  labels:
    severity: critical
  annotations:
    summary: "检测到非法出牌，立即回滚"
    action: "自动回滚到上一版本Champion"

- alert: SystemCrashDetected
  expr: sum(rate(game_crashes_total[5m])) > 0
  for: 0m
  labels:
    severity: critical
  annotations:
    summary: "AI崩溃，立即回滚"
```

### 6.2 P1告警（紧急处理）

```yaml
- alert: LatencyP99Exceeded
  expr: histogram_quantile(0.99, decision_latency_ms) > 150
  for: 15m
  labels:
    severity: high
  annotations:
    summary: "P99延迟超过150ms持续15分钟"
    action: "检查系统负载，考虑降级"

- alert: WinRateDrift
  expr: abs(winrate_7d - winrate_baseline) > 0.05
  for: 1h
  labels:
    severity: high
  annotations:
    summary: "胜率漂移超过5%"
    action: "检查策略是否异常"
```

### 6.3 P2告警（关注）

```yaml
- alert: BlunderRateIncreased
  expr: blunder_rate_24h > blunder_rate_baseline * 1.2
  for: 6h
  labels:
    severity: warning
  annotations:
    summary: "明显错误率上升20%"
    action: "分析决策日志，考虑重新训练"
```

## 8. 指标使用指南

### 8.1 日常运营检查清单

**每日检查（5分钟）**:
- [ ] L1红线指标全绿
- [ ] 无P0/P1告警
- [ ] 胜率波动<2%

**每周检查（30分钟）**:
- [ ] L2用户体验指标在目标范围
- [ ] 难度梯度健康
- [ ] 明显错误率无上升趋势

**每月检查（2小时）**:
- [ ] L3进化质量指标健康
- [ ] 累计提升符合预期
- [ ] 生成月度报告

### 8.2 异常诊断流程

```
发现指标异常
    ↓
[Step 1] 确认影响范围（单难度/全局？）
    ↓
[Step 2] 查看关联指标（延迟/错误率/胜率）
    ↓
[Step 3] 回溯变更历史（最近晋升/配置变更？）
    ↓
[Step 4] 分析决策日志（采样100局）
    ↓
[Step 5] 决策：回滚/修复/观察
```

### 8.3 指标改进优先级

**P0优先级**（立即处理）:
- 合法性率<100%
- 崩溃率>0%
- P99延迟>200ms

**P1优先级**（本周处理）:
- 明显错误率>5%
- 胜率漂移>5%
- 完局率<80%

**P2优先级**（本月处理）:
- 策略多样性<2.0
- 接近局比率<30%
- 晋升成功率<15%
