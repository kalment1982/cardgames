# AI共享决策输入定义 v1.0

## 1. 文档定位

本文档是当前项目 `RuleAI` 与 `PPO AI` 的共享决策输入定义文档。

用途只有 3 个：

1. 统一 AI 决策输入字段的语义
2. 统一阶段 1 / 阶段 2 的输入边界
3. 统一 RuleAI 上下文到 PPO observation 的映射口径

本文档不负责描述：

1. RuleAI 的策略逻辑
2. PPO 的训练流程
3. 奖励函数
4. 联赛、评估与上线流程

这些内容应分别留在：

- `doc/规则AI架构设计_v2.1.md`
- `doc/PPO训练架构设计_v1.0.md`

共享输入项的具体编码、归一化与 PPO observation 拼接方式，以：

- [AI共享决策输入编码定义_v1.0](./AI共享决策输入编码定义_v1.0.md)

为准。

---

## 2. 使用规则

本项目后续所有 agent 在处理 AI 决策输入相关问题时，必须遵守以下规则：

1. 若字段语义、阶段归属、输入边界发生变化，先更新本文档，再更新 RuleAI / PPO 文档或代码
2. RuleAI 与 PPO 文档应引用本文档，不应各自重复定义同一字段语义
3. 若代码中的 `RuleAIContext / HandProfile / DecisionFrame / MemorySnapshot / InferenceSnapshot` 字段发生新增、删除或含义变化，应同步回写本文档
4. 若 PPO observation 与本文档映射不一致，应优先修正文档或映射实现，不允许长期双口径并存

## 4. 阶段1必需输入表

阶段 1 原则：

- 只保留当前出牌决策直接需要的信息
- 不引入复杂记牌、概率推断和高层风险摘要
- 目标是低噪声、强相关、可直接支撑 `PlayTricks` 决策

| 共享输入项 | 文字描述 | 为什么阶段1必须有 | 为什么不属于高噪声 |
|---|---|---|---|
| 自己手牌 | 当前玩家手里实际持有的全部牌 | 决定“我能出什么牌” | 原始真值信息 |
| 自己角色 | 当前玩家属于庄家方还是闲家方 | 决定策略目标方向 | 原始真值信息 |
| 自己座位 | 当前玩家的座位编号 | 决定相对位置与出牌顺序 | 原始真值信息 |
| 庄家座位 | 本局庄家的座位编号 | 决定阵营关系与队友位置 | 原始真值信息 |
| 主花色/无主 | 本局主牌体系 | 决定主副牌关系和压制规则 | 规则真值信息 |
| 当前级牌 | 本局级牌等级 | 决定关键牌牌力 | 规则真值信息 |
| 当前第几墩 | 当前整局进度 | 区分前中后期 | 低噪声进度信息 |
| 当前出牌位置 | 当前玩家在本墩的出牌顺序 | 决定首发/跟牌策略差异 | 低噪声顺序信息 |
| 当前墩首发牌 | 本墩第一手牌 | 决定跟牌约束与牌型体系 | 当前牌面真值 |
| 当前领先牌组 | 当前暂时最大的那组牌 | 决定能否压过当前赢家 | 当前牌面真值 |
| 当前赢家 | 当前这墩谁在赢 | 决定抢/让/送分判断 | 当前牌面真值 |
| 队友是否暂时领先 | 当前赢家是否为队友 | 是高价值策略压缩信息 | 已被 RuleAI 证明有效 |
| 当前墩累计分数 | 当前这一墩已出现的总分 | 决定是否值得抢这一墩 | 当前收益真值 |
| 主牌数量 | 自己剩余主牌总数 | 决定主牌控制能力 | 低噪声结构统计 |
| 高主数量 | 自己高控制主牌的数量 | 决定是否能稳住关键墩 | 低噪声结构统计 |
| 王数量 | 自己持有大小王数量 | 反映最高控制牌储备 | 低噪声结构统计 |
| 级牌数量 | 自己持有级牌数量 | 反映关键牌储备 | 低噪声结构统计 |
| 主牌对子数量 | 自己主牌对子结构数量 | 影响控牌和结构打法 | 低噪声结构统计 |
| 主牌拖拉机数量 | 自己主牌拖拉机结构数量 | 影响结构强度和控制力 | 低噪声结构统计 |
| 各花色长度 | 自己各副花色剩余长度 | 决定长短门与跑副牌能力 | 低噪声结构统计 |
| 手牌分牌数量 | 自己手里还剩多少分牌 | 决定送分/丢分风险 | 低噪声收益统计 |

## 5. 阶段1明确不引入表

| 输入项 | 文字描述 | 为什么阶段1不引入 |
|---|---|---|
| 已出牌计数 | 每张牌已经出过多少张 | 属于记牌信息，阶段2统一引入 |
| 各玩家缺门信息 | 某玩家已确定没有某门花色 | 属于观察推导结果，不是阶段1最小必需输入 |
| 无对子证据 | 某玩家在某体系下缺少对子能力的证据 | 属于结构推断，会增加噪声 |
| 无拖拉机证据 | 某玩家在某体系下缺少拖拉机能力的证据 | 属于更高层结构推断 |
| 已知底牌信息 | 已明确知道的底牌内容 | 阶段1不训练保底/抠底策略 |
| 其他玩家主牌数量估计 | 其他玩家剩余主牌数量的估计范围 | 属于推断信息，阶段2再加入 |
| 其他玩家高主风险 | 其他玩家仍持高主的风险等级 | 属于概率/风险摘要 |
| 某体系首发被毙风险 | 继续首发某体系的被毙风险 | 属于高层安全判断 |
| 保底压力 | 当前局面对保底是否有明显压力 | 属于高层语义摘要 |
| 掉庄压力 | 当前是否存在明显掉庄风险 | 属于高层语义摘要 |
| 抠底争夺压力 | 当前是否值得冲击抠底 | 属于高层语义摘要 |
| 分数压力 | 当前比分是否紧张 | 属于高层语义摘要 |
| 末局阶段 | 当前是否进入末局及其紧迫程度 | 属于后期专项语义 |
| 底牌分数 | 底牌分值总量 | 阶段1不聚焦保底/抠底收益 |

## 6. 阶段2新增输入表

阶段 2 原则：

- 在阶段 1 基础上补充记牌、推断和局势风险
- 目标是让 AI 从“会打牌”进化到“会看局势”

| 共享输入项 | 文字描述 | 为什么放到阶段2 | 主要支撑能力 |
|---|---|---|---|
| 已出牌计数 | 每一种牌在牌局中已经出现了多少张 | 这是记牌基础，但阶段1可先不引入 | 判断剩余未知牌 |
| 各玩家缺门信息 | 哪些玩家已经确定没有某门花色 | 这是最关键记牌信息，但不属于阶段1最小集 | 判断是否可能被毙、是否可继续首发某门 |
| 已知底牌信息 | 当前已明确知道的底牌内容 | 只有阶段2开始才考虑保底/抠底 | 保底/抠底收益判断 |
| 无对子证据 | 从跟牌行为推断某玩家缺少对子能力 | 属于结构证据，阶段1过早引入噪声大 | 对子安全性、结构判断 |
| 无拖拉机证据 | 从行为推断某玩家缺少拖拉机能力 | 属于结构证据，适合阶段2 | 甩牌与结构型首发判断 |
| 其他玩家主牌数量估计 | 对其他玩家主牌剩余量的估计范围 | 属于推断信息，阶段2再加入 | 主牌控制与末局控制 |
| 其他玩家高主风险 | 其他玩家仍持有高主控制牌的风险等级 | 属于稳赢/稳不住判断 | 高分墩保护、主牌控制 |
| 某体系首发被毙风险 | 继续首发某花色/体系的被毙风险 | 属于高层安全评估 | 首发安全性判断 |
| 保底压力 | 当前局面对庄家方保底是否有压力 | 需配合阶段2保底策略 | 庄家保底 |
| 掉庄压力 | 当前是否存在明显掉庄风险 | 阶段2才开始显式引入 | 庄家方风险控制 |
| 抠底争夺压力 | 当前是否值得争取抠底 | 阶段2才开始显式引入 | 闲家冲击抠底 |
| 分数压力 | 当前比分是否紧张 | 这是局势摘要，阶段2才加入 | 高压局面抢分/止损 |
| 末局阶段 | 当前是否进入末局及其紧迫程度 | 阶段2才开始引入末局策略 | 最后几墩控制 |
| 底牌分数 | 当前底牌总分 | 阶段2才需要显式使用 | 保底/抠底收益量化 |
