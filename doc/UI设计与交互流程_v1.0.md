## 1. 整体布局设计

### 1.1 目标设备规格

**iPhone 15 Pro Max**：
- 屏幕尺寸：6.7英寸
- 分辨率：2796 × 1290 像素（横屏）
- 像素密度：460 PPI
- 安全区域：需考虑刘海和圆角

**华为 Nova 14**：
- 屏幕尺寸：6.88英寸
- 分辨率：2412 × 1084 像素（横屏）
- 像素密度：~400 PPI
- 安全区域：需考虑打孔屏

**设计基准**：
- 基准分辨率：2400 × 1080（横屏）
- 适配策略：等比缩放 + 安全区域适配

### 1.3 详细布局说明

#### 1.3.1 左上角信息区（双行显示）

```
┌─────────────────────────────┐
│ 我方｜5｜♠｜抓分            │  ← 第一行
│ 对方｜3｜  ｜120             │  ← 第二行
└─────────────────────────────┘
```

**第一行（我方）**：
- "我方"：队伍标识
- "5"：我方当前级别（直接显示数字，不显示"打"字）
- "♠"：当前主花色（如果是我方亮的主）
  - 如果无人亮主，显示"NT"（No Trump）
- "抓分"：固定文本标签

**第二行（对方）**：
- "对方"：对方队伍标识
- "3"：对方当前级别（直接显示数字）
- 空白或花色：主花色位置（如果是对方亮的主则显示）
  - 如果无人亮主，显示"NT"（No Trump）
- "120"：当前抓分数值（对齐"抓分"位置）

**示例**：
```
无主局：
我方｜5｜NT｜抓分
对方｜3｜  ｜120

我方亮主：
我方｜5｜♠｜抓分
对方｜3｜  ｜120

对方亮主：
我方｜5｜  ｜抓分
对方｜3｜♥｜120
```

**说明**：
- 只显示一个抓分统计（闲家得分）
- 第一行显示"抓分"标签
- 第二行对应位置显示具体分数
- 不显示墩数（第X/25墩）

#### 1.3.2 玩家位置布局（头像+名字）

**上方（北）**：
```
    [👤 北]
```

**左侧（西）**：
```
[👤 西]
```

**右侧（东）**：
```
[👤 东]
```

**下方（南-玩家）**：
- 无需显示头像（手牌区即代表玩家）
- 或在手牌区上方显示小头像

**说明**：
- 只显示方位名称（南、北、东、西）
- 不显示"队友"、"对手"标识
- 不显示手牌数量（所有人都一样）

#### 1.3.3 出牌显示区（按位置分布）

**上家（北）的出牌**：
- 位置：中央偏上
- 排列：从左到右水平排列
- 示例：`🂡 🂢 🂣`

**左家（西）的出牌**：
- 位置：左侧中央，向中间延伸
- 排列：从右向左（靠近中央）
- 示例：`🂠 🂠 🂠`

**右家（东）的出牌**：
- 位置：右侧中央，向中间延伸
- 排列：从左向右（靠近中央）
- 示例：`🂠 🂠 🂠`

**下家（南-玩家）的出牌**：
- 位置：手牌区上方
- 排列：从左到右水平排列
- 示例：`🂤 🂥 🂦`

#### 1.3.4 亮牌框（亮牌阶段显示）

位置：手牌区正上方

```
┌────────────────────────────────┐
│ [亮主框] 🃏 ♠ ♥ ♣ ♦            │
│          ↑  ↑ ↑ ↑ ↑            │
│         王 黑红梅方              │
└────────────────────────────────┘
```

**功能**：
- 显示5个亮主选项：王、♠、♥、♣、♦
- 可亮主的花色：高亮显示（彩色+发光）
- 不可亮主的花色：灰色半透明
- 点击高亮的花色 → 亮主/反主

**显示时机**：
- 仅在发牌阶段显示
- 发牌过程中，如果手中有级牌，对应花色高亮
- 发牌结束后立即消失（关闭亮主窗口）
- 出牌阶段不显示

#### 1.3.5 手牌区

位置：屏幕下方，占据约1/3高度

```
┌────────────────────────────────┐
│  🂡 🂢 🂣 🂤 🂥 🂦 🂧 🂨 🂩 🂪    │

**特性**：
- 自动排序（主牌在前，副牌在后）
- 扇形展开（可选）或平铺
- 选中的牌上移突出显示
- 支持智能选择（拖拉机、对子）

#### 1.3.6 操作按钮区

位置：手牌区下方

```
[出牌] [扣底] [查看底牌] [上一轮牌]
```

**按钮说明**：
- **出牌**：确认出牌（选中牌后激活）
- **扣底**：扣底阶段显示（仅庄家）
- **查看底牌**：查看已扣的底牌（仅庄家，出牌阶段可用）
- **上一轮牌**：查看上一墩的出牌记录

**按钮状态**：
- 可用：绿色/蓝色
- 不可用：灰色
- 当前不显示：隐藏

#### 1.3.7 设置按钮

位置：左上角（信息区左侧）

```
[⚙️]  我方｜打5｜♠｜30分
     对方｜打3｜  ｜120分
```

**功能**：
- 点击打开设置菜单
- 设置选项：音效、动画速度、规则说明、退出游戏

## 2. 游戏阶段与界面

### 2.1 阶段流程图

```
开始游戏
   ↓
[1. 发牌阶段] → 发牌过程中可以亮主
   ↓           （发牌结束立即关闭亮主窗口）
   ↓
[2. 扣底阶段] → 如果玩家是庄家，选择8张扣底
   ↓
[3. 出牌阶段] → 出牌循环（直到手牌出完）
   ↓
[4. 结算阶段] → 显示得分和升级结果
   ↓
下一局 / 结束游戏
```

**说明**：
- 亮主不是独立阶段，而是在发牌过程中进行
- 发牌结束后立即关闭亮主窗口，进入扣底阶段
- 如果发牌结束后无人亮主，则本局为"无主"（无将）
- 出牌阶段的墩数不固定，取决于每墩出牌的张数
  - 如果每墩都出单张，则25墩
  - 如果出对子、拖拉机、甩牌，墩数会减少

### 2.2 各阶段界面设计

#### 2.2.1 发牌阶段（含亮主）

**界面元素**：
- 中央显示"正在发牌..."
- 卡牌从牌堆飞向各个玩家位置
- **亮牌框**：手牌上方显示 🃏 ♠ ♥ ♣ ♦
- 左上角信息栏显示当前级别
- 如果有人亮主，显示当前主花色

**亮主交互**（发牌过程中）：
- 每发一张牌，玩家手牌自动排序
- 如果手中有级牌，亮牌框对应花色高亮
- 点击高亮的花色图标 → 亮主
- AI亮主时显示提示："东 亮主♥"
- 可以反主（如果有更高优先级的牌型）

**发牌结束**：
- 发牌结束后，亮牌框立即消失
- 如果无人亮主，显示提示："本局无主"（2秒）
- 进入扣底阶段

**交互**：
- 发牌过程中可以亮主
- 可选：加速按钮（跳过动画）

#### 2.2.2 扣底阶段（仅庄家）

**界面元素**：
- 顶部提示："你是庄家，请选择8张牌扣底"
- 手牌区显示33张牌（25+8底牌）
- 底牌用特殊边框标识
- 显示已选择的牌数："已选择 X/8 张"

**交互**：
- 点击牌 → 选中/取消选中（切换状态）
- 选中的牌显示高亮边框
- 选够8张后，"确认扣底"按钮激活
- 点击"确认扣底" → 进入出牌阶段

**扣底界面**：
```
┌─────────────────────────────────────┐
│  请选择8张牌扣底 (已选择 5/8)        │
│                                      │
│  底牌: 🂡 🂢 🂣 🂤 🂥 🂦 🂧 🂨         │
│                                      │
│  手牌: 🂩 🂪 🂫 🂬 🂭 🂮 ...          │
│                                      │
│        [确认扣底] [重新选择]         │
└─────────────────────────────────────┘
```

#### 2.2.3 出牌阶段

**界面元素**：
- 左上角信息栏显示当前抓分
- 四个方位显示玩家头像
- 出牌显示在头像旁边
- 手牌区显示玩家手牌
- 操作按钮：出牌、查看底牌、上一轮牌

**出牌规则**：
- 首家可以出任意合法牌型（单张、对子、拖拉机、甩牌等）
- **跟牌必须出与首家相同张数的牌**
- 跟牌必须遵循跟牌规则（有首引花色必须跟，有对子/拖拉机必须跟等）

**交互**：

**A. 玩家出牌（首家）**：
1. 提示："轮到你出牌（首家）"
2. 点击手牌选择要出的牌（智能选择，可多选）
3. 选中的牌上移显示
4. 点击"出牌"按钮 或 向上拖拽出牌
5. 出牌后，牌飞到手牌区上方（南的出牌位置）

**B. 玩家跟牌**：
1. 提示："轮到你跟牌（需出X张）"
2. 不合法的牌变灰（不可选）
3. 合法的牌可点击选择（智能选择）
4. **必须选择与首家相同张数的牌**
5. 点击"出牌"按钮 或 向上拖拽出牌

**C. AI出牌**：
1. AI头像显示黄色发光边框（轮到AI）
2. 显示"X 思考中..."（1秒）
3. AI的牌飞到对应位置
4. 显示AI出的牌

**D. 一墩结束**：
1. 高亮显示赢家的牌（绿色发光边框）
2. 显示提示："X 赢得本墩（+10分）"
3. 更新抓分显示
4. 1秒后所有牌飞向赢家（收牌动画）
5. 清空出牌区
6. 进入下一墩

**E. 游戏结束**：
- 当所有玩家手牌出完（手牌数为0）时，游戏结束
- 进入结算阶段

**说明**：
- 墩数不固定，取决于每墩出牌的张数
- 如果每墩都出单张，则25墩
- 如果出对子、拖拉机、甩牌，墩数会减少

#### 2.2.4 结算阶段

**界面元素**：
- 大标题："本局结束"
- 显示最终得分
- 显示获胜方
- 显示升级情况
- 显示抠底得分（如果有）

**交互**：
- 点击"下一局"按钮 → 开始新一局
- 点击"查看详情"按钮 → 显示详细统计

**结算界面**：
```

│  庄家得分: 80 分                     │

## 3. 交互流程详细说明

### 3.1 手牌操作交互（核心功能）

#### 3.1.1 自动排序规则

**排序优先级**：
1. **主牌在前**：大王 > 小王 > 主级牌 > 副级牌 > 主花色牌
2. **副牌在后**：按花色分组（♠ ♥ ♣ ♦），每组内按点数排序
3. **同点数牌**：相邻排列（便于识别对子和拖拉机）

**示例**（打2，主花色♠）：
```
主牌区: 大🃏 小🃏 ♠2 ♠2 ♥2 ♣2 ♦2 ♠A ♠A ♠K ♠K ...
副牌区: ♥A ♥A ♥K ♥K ... ♣A ♣A ♣K ... ♦A ♦A ♦K ...
```

**排序时机**：
- 发牌后自动排序
- 出牌后自动重排
- 亮主后重新排序（主花色变化）

#### 3.1.2 智能选择功能

**功能说明**：
点击一张牌时，系统自动识别并选择相关的对子或拖拉机。

**选择逻辑**：

**情况1：点击单张牌（无对子/拖拉机）**
- 结果：仅选中该张牌
- 示例：点击 ♥A → 选中 ♥A

**情况2：点击对子中的一张**
- 结果：自动选中整个对子
- 示例：点击 ♠K（有两张♠K相邻）→ 选中 ♠K ♠K

**情况3：点击拖拉机中的一张**
- 结果：自动选中整个拖拉机
- 示例：点击 ♠K（有 ♠K♠K ♠Q♠Q 拖拉机）→ 选中 ♠K ♠K ♠Q ♠Q

**情况4：点击已选中的牌**
- 结果：智能取消选择
- 如果是对子的一部分 → 取消整个对子
- 如果是拖拉机的一部分 → 取消整个拖拉机
- 如果是单张 → 取消该张

**智能识别规则**：
1. 优先识别最长的拖拉机
2. 如果不是拖拉机，识别对子
3. 如果都不是，识别单张

**示例场景**：

**场景A：选择对子**
```
手牌: ♠A ♠A ♠K ♠Q ♠J
操作: 点击第一张♠A
结果: ♠A ♠A 被选中（上移）
```

**场景B：选择拖拉机**
```
手牌: ♠K ♠K ♠Q ♠Q ♠J ♠J
操作: 点击第一张♠K
结果: ♠K ♠K ♠Q ♠Q ♠J ♠J 全部被选中（上移）
```

**场景C：取消拖拉机**
```
已选中: ♠K ♠K ♠Q ♠Q ♠J ♠J（上移状态）
操作: 点击任意一张（如♠Q）
结果: 整个拖拉机取消选中（下移）
```

**场景D：混合选择**
```
手牌: ♠K ♠K ♠Q ♠Q ♥A ♥A ♣10
操作1: 点击♠K → 选中 ♠K ♠K ♠Q ♠Q（拖拉机）
操作2: 点击♥A → 追加选中 ♥A ♥A（对子）
操作3: 点击♣10 → 追加选中 ♣10（单张）
结果: 共选中8张牌
```

#### 3.1.3 出牌方式（双模式）

**方式1：点击"出牌"按钮**
1. 选择牌（点击，牌上移）
2. 点击"出牌"按钮
3. 验证合法性
4. 出牌动画

**方式2：向上拖拽出牌**
1. 选择牌（点击，牌上移）
2. 向上拖拽选中的牌
3. 拖拽超过阈值（50px）→ 自动出牌
4. 验证合法性
5. 出牌动画

**拖拽交互细节**：
- 拖拽时，选中的牌跟随手指移动
- 拖拽距离 < 50px：松手后牌回到原位
- 拖拽距离 ≥ 50px：松手后自动出牌
- 拖拽过程中显示虚线提示："松手出牌"

#### 3.1.4 选中状态视觉反馈

**正常状态**：
- 位置：基准线
- 边框：无
- 阴影：轻微阴影

**选中状态**：
- 位置：向上移动 20px
- 边框：黄色发光边框（2px）
- 阴影：增强阴影

**不可选状态**（跟牌时）：
- 位置：基准线
- 蒙版：灰色半透明（50%）
- 边框：无

**动画**：
- 选中/取消：0.2秒缓动动画
- 缓动函数：ease-in-out

### 3.2 出牌交互流程

#### 流程图

```
玩家回合开始
   ↓
判断：首家 or 跟牌？
   ↓
┌──────────┴──────────┐
│                     │
首家                 跟牌
│                     │
所有牌可选           不合法的牌变灰
│                     │
└──────────┬──────────┘
           ↓
    玩家选择牌（智能选择）
           ↓
    点击"出牌" 或 向上拖拽
           ↓
    验证合法性
           ↓
    ┌─────┴─────┐
    │           │
   合法        不合法
    │           │
   出牌      显示错误提示
    │           │
    ↓           └→ 重新选择
  下一家
```

#### 详细步骤

**步骤1：轮到玩家**
- 显示提示："轮到你出牌"
- 手牌区激活（可点击）
- 如果是跟牌，不合法的牌变灰

**步骤2：选择牌（智能选择）**
- 点击牌 → 智能选择对子/拖拉机，牌上移
- 再次点击 → 智能取消选择，牌下移
- 可追加选择其他牌
- 显示已选择的牌数

**步骤3：确认出牌**
- **方式A**：点击"出牌"按钮
- **方式B**：向上拖拽选中的牌
- 验证合法性
- 如果不合法，显示错误提示：
  - "必须跟首引花色"
  - "有对子必须出对子"
  - "有拖拉机必须出拖拉机"
- 如果合法，牌飞到出牌区

**步骤4：等待其他玩家**
- 显示"等待其他玩家..."
- AI自动出牌（1秒延迟）

**步骤5：判定赢家**
- 4家都出牌后，高亮赢家的牌
- 显示提示："X 赢得本墩"
- 如果有分牌，显示得分

## 6. 动画效果

### 6.1 发牌动画

**动画描述**：

- 按顺序发牌（南→东→北→西循环）

- 加速模式：0.1秒/张
- 缓动函数：ease-out
- 轨迹：直线飞行

**实现细节**：
1. 牌堆位置：屏幕中央
2. 每张牌依次飞向目标玩家
3. 到达后翻转显示（玩家手牌）或保持背面（AI手牌）
4. 玩家手牌自动排序

### 6.2 出牌动画

**动画描述**：
- 卡牌从手牌区飞到出牌区对应位置

**参数**：
- 飞行时间：0.5秒
- 缓动函数：ease-out
- 轨迹：抛物线（可选）或直线

**实现细节**：
1. 选中的牌从手牌区起飞
2. 飞向出牌区对应位置
3. 到达后放大显示（60px → 80px）
4. 手牌区自动重排

### 6.3 赢墩动画

**动画描述**：
- 赢家的牌发光高亮
- 所有牌飞向赢家（收牌）

**参数**：
- 高亮持续：1秒
- 收牌飞行：0.5秒
- 缓动函数：ease-in

**实现细节**：
1. 赢家的牌显示绿色发光边框
2. 显示提示："X 赢得本墩（+10分）"
3. 1秒后，所有牌飞向赢家
4. 牌消失，清空出牌区
5. 进入下一墩

**动画描述**：
- 卡牌上移/下移

**参数**：
- 移动距离：20px
- 动画时间：0.2秒
- 缓动函数：ease-in-out

**实现细节**：
1. 点击牌 → 牌向上移动 20px
2. 同时显示黄色边框

- 回弹时间：0.3秒（未达到阈值）

**实现细节**：

3. 显示虚线提示："松手出牌"

## 13. 关键规则说明

### 跟牌规则
1. **张数约束**：跟牌必须出与首家相同张数的牌
2. **花色约束**：有首引花色必须先跟该花色
3. **牌型约束**：
   - 首引对子，有对子必须出对子
   - 首引拖拉机，有拖拉机必须出拖拉机
4. **垫牌规则**：无首引花色时可垫牌或毙牌

### 墩数说明
- 墩数不固定，取决于每墩出牌的张数
- 示例：
  - 每墩出1张 → 25墩
  - 每墩出2张（对子）→ 12-13墩
  - 每墩出4张（拖拉机）→ 6-7墩
  - 混合出牌 → 墩数不定

### 无主局
- 发牌结束后无人亮主 → 无主局
- 无主时：仅大小王和级牌为主牌，无主花色
- 左上角主花色位置显示"NT"（No Trump）
