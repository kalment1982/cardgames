# 规则AI甩牌策略 v1.0

## 版本信息
- 日期：2026-03-14
- 状态：已落地到规则AI代码（`CardMemory + AIPlayer`）

## 目标
- 在首引甩牌前，按“同花色/同主牌体系”做保守判定。
- 满足“任一其他玩家可拦截即视为甩牌不安全”。
- 在硬判定基础上保留概率估计能力，用于难度分层与后续调优。

## 核心口径（与你确认一致）
1. 判定对象是“其他三家中的任意一家”，不是仅对家或仅敌方两家。
2. 单牌逐张检查，且只看同花色同体系压制；副牌被主牌毙不计入甩牌拦截判定。
3. 对子可使用历史硬证据：若某花色（或主牌体系）曾首发对子且其余三家都未跟对子，则可判该三家在该体系无对子压制能力。
4. 甩牌中每个对子独立检查是否可被更大同体系对子拦截。
5. 拖拉机按“同体系、同长度、最高对更大”判断可拦截。
6. 主牌统一按“主牌体系”处理（大小王、级牌、主花色牌），不按单一花色拆开。
7. 保留概率估计，用于策略阈值；但先走硬判定。
8. 先做“总池保守判定”：只要剩余总池存在可拦截组合，就判为有风险。

## 决策流程
1. 构建未知总池：
   - 总牌池（两副）- 已出牌 - 自己手牌 -（庄家可见）已知底牌。
2. 拆解甩牌组件：
   - 拖拉机优先 -> 对子 -> 单牌（与 `ThrowValidator` 统一）。
3. 组件硬判定：
   - 单牌：未知总池是否存在同体系更大单牌。
   - 对子：未知总池是否存在同体系更大对子；若“其他玩家全体已有无对子证据”则直接安全。
   - 拖拉机：未知总池是否存在同体系同长度且最高对更大的拖拉机；若“其他玩家全体无对子证据”则直接安全。
4. 任一组件可拦截 => 本次甩牌“硬判定不安全”。
5. 概率估计：
   - 在“可拦截”前提下，结合可出该体系的玩家数、阻断密度估计成功率。
6. AI决策：
   - 中/高/专家难度先要求“硬判定安全”，再比较概率阈值（0.6/0.7/0.8）。

## 示例说明
### 示例A：副牌单张
- 甩牌含 `♠10`（副牌）。
- 其他玩家若存在 `♠J/♠Q/♠K/♠A` 可拦截，则该单牌组件不安全。
- 其他玩家用主牌毙 `♠10` 不计入本条“甩牌可拦截”判定。

### 示例B：对子
- 甩牌含 `♥99`。
- 若历史上出现过“♥花色首发对子”，且其他三家均未跟对子，可将 `♥` 体系对子视为无压制能力。
- 否则按未知总池检查是否仍有更大 `♥` 对子可组成（如 `♥JJ`）。

### 示例C：拖拉机
- 甩牌含 `♣88♣77`（2对拖拉机）。
- 只有“同体系、同长度”的拖拉机可拦截，例如 `♣QQ♣JJ`。
- 若只能组成更短拖拉机或仅有单对子，不算该组件拦截成功。
