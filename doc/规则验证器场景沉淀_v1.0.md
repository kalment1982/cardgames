# 规则验证器场景沉淀 v1.0

- 文档状态：可执行流程
- 适用范围：规则 AI / 跟牌校验 / 赢家判定 / 甩牌判定回归
- 目标：把一次真实问题牌局，快速沉淀成可重复执行的自动化验证

---

## 1. 输入来源

优先使用 AI 决策 bundle：

- 路径：`logs/decision/YYYY-MM-DD/<round_id>/<trick_id>/<decision_trace_id>.json`
- 关键优势：
  - 自带 `my_hand`
  - 自带 `lead_cards` / `current_winning_cards`
  - 自带 `dealer_index` / `player_index` / `cards_left_min`
  - 自带 `defender_score` / `bottom_points`
  - 比 replay markdown 更容易精确还原上下文

## 2. 沉淀目标分三层

### 2.1 规则层

适用问题：

- 跟牌是否合法
- 一墩赢家是否正确
- 甩牌是否应成功

对应测试：

- `FollowValidatorRegressionTests`
- `TrickJudgeRegressionTests`
- `TrickJudgeTrumpCutMixedThrowTests`
- `ThrowRulesTests`

结论：

- 这层负责验证“规则本身对不对”
- 不关心 AI 有没有想到这手牌

### 2.2 候选层

适用问题：

- 明明存在可赢牌，但候选列表没生成出来
- 甩牌候选缺失
- 跟牌候选太少，导致意图解析失真

对应测试：

- `tests/V21/FollowCandidateGeneratorTests.cs`
- `tests/V21/LeadCandidateGeneratorTests.cs`

结论：

- 这层负责验证“AI 有没有看到正确候选”

### 2.3 决策层

适用问题：

- 候选是对的，但最终选牌不合理
- 意图判断错误
- 高分墩不抢、队友已赢仍乱压、终局不保底

对应测试：

- `tests/V21/FollowPolicy2Tests.cs`
- `tests/V21/LeadPolicy2Tests.cs`
- `tests/V21/ScenarioBenchmarkTests.cs`
- `tests/V21/DecisionBundleScenarioFactoryTests.cs`

结论：

- 这层负责验证“AI 最终有没有做出合理选择”

## 3. 标准沉淀流程

### 第一步：复制 bundle 为固定夹具

命令：

```bash
./scripts/add_decision_bundle_fixture.sh \
  logs/decision/2026-03-15/round_xxx/trick_xxx/decision_xxx.json \
  my_case_name
```

产物：

- `tests/V21/Fixtures/decision_bundles/my_case_name.json`

### 第二步：从 bundle 生成场景

使用：

- `DecisionBundleScenarioFactory.FromBundleFile(...)`

工厂会自动还原：

- 配置：主花色、级牌
- 角色：庄家/庄家队友/闲家
- 上下文：手牌、首引、当前赢家、当前墩分、剩余手牌数、底牌分压力

### 第三步：写断言

推荐优先写“行为断言”，而不是一上来就写死唯一牌组：

- `ExpectIntent(...)`
- `ExpectBeatCurrentWinner(...)`
- `ExpectAllTrump(...)`
- `ExpectNotExactCards(...)`
- `ExpectCount(...)`

只有在规则要求唯一解时，再补：

- `ExpectExactCards(...)`

### 第四步：必要时补规则层/候选层测试

判断标准：

- 如果问题是“这手按规则就不合法”：
  - 补规则层测试
- 如果问题是“规则允许，AI 也应该想到，但候选没生成”：
  - 补候选层测试
- 如果问题是“候选已有，但最终决策不对”：
  - 补决策层测试

## 4. 当前推荐模板

```csharp
var parsed = DecisionBundleScenarioFactory.FromBundleFile(
    "tests/V21/Fixtures/decision_bundles/my_case_name.json");

var scenario = DecisionBundleScenarioFactory.FromBundleFile(
    "tests/V21/Fixtures/decision_bundles/my_case_name.json",
    name: "Follow_MyCase",
    description: "描述这手牌为什么错",
    expectations: new IScenarioExpectation[]
    {
        new ExpectCount(2),
        new ExpectIntent(DecisionIntentKind.TakeScore),
        new ExpectAllTrump(parsed.Config),
        new ExpectBeatCurrentWinner(parsed.Config, parsed.CurrentWinningCards)
    });

var result = ScenarioJudge.Run(scenario);
Assert.True(result.Passed, result.ToString());
```

## 5. 建议约束

### 建议1：每个线上问题至少沉淀 1 条决策层回归

- 避免同类问题反复出现

### 建议2：如果根因是“规则/候选缺失”，必须再补 1 条更底层测试

- 决策层只能看到结果
- 底层测试才能精准卡住根因

### 建议3：场景命名要包含意图

推荐格式：

- `Follow_Trick0012_OvercutHighScore`
- `Lead_Endgame_SafeThrowCloseout`
- `Follow_PartnerWinning_SendPointsWithoutBurningAce`
